# Data Models Design Document

## Overview

The `models.py` module provides the core `Task` dataclass for the Todo Manager CLI. It enforces data validation, type safety, and provides serialization/deserialization methods for seamless integration with the database and export layers.

## Design Decisions

### 1. **Dataclass-based Implementation**

We use Python's `@dataclass` decorator because it:
- Automatically generates `__init__`, `__repr__`, and `__eq__` methods
- Provides type hints for better IDE support and static analysis
- Requires less boilerplate code than traditional classes
- Works naturally with our validation approach via `__post_init__`

### 2. **Post-Initialization Validation**

Validation occurs in `__post_init__()` rather than in `__init__()`:
- **Benefit**: Leverages dataclass's own initialization process
- **Safety**: Ensures all fields are set before validation runs
- **Error Handling**: Raises `ValidationError` with clear messages

### 3. **Priority and Category as Strings**

While we could use Enum types:
```python
from enum import Enum
class Priority(Enum):
    LOW = "Low"
    ...
```

We chose plain strings because:
- Simpler serialization to/from JSON and database
- More flexible for future priority levels without code changes
- Easier to work with in CLI (users type "Low" not "Priority.LOW")
- `VALID_PRIORITIES` set provides type-like validation

### 4. **Timestamp Handling**

- **Created_at**: Immutable, set only at creation (default: `datetime.now()`)
- **Updated_at**: Updated whenever the task is modified
- **ISO Format Storage**: Timestamps stored as ISO format strings in JSON/DB for portability
- **Datetime Objects**: Kept as Python datetime objects in memory for type safety and operations

### 5. **Immutable-by-Convention**

The dataclass is frozen-safe (though not explicitly frozen to allow updates via database methods):
- Fields shouldn't be modified directly in client code
- All mutations happen through `Database.update_task()` to maintain consistency
- This ensures database updates always cascade properly

## API Surface

### Validation Methods

```python
Task._validate() -> None
```
- Called automatically after initialization
- Validates each field against type and value constraints
- Raises `ValidationError` with descriptive messages

### Serialization Methods

```python
to_dict() -> dict
to_json() -> str

@classmethod
from_dict(data: dict) -> Task

@classmethod
from_json(json_str: str) -> Task
```

All conversions properly handle:
- Type conversion (datetime <-> ISO string)
- Validation of reconstructed objects
- Clear error messages on failure

## Field Specifications

| Field | Type | Required | Constraints | Notes |
|-------|------|----------|-------------|-------|
| id | int | Yes | > 0 | Auto-generated by database |
| title | str | Yes | Non-empty | User-visible task name |
| description | str | Yes | Any | Can be empty string, but field must exist |
| priority | str | Yes | Low/Medium/High | Validated against VALID_PRIORITIES |
| category | str | Yes | Non-empty | User-defined grouping |
| completed | bool | No | True/False | Default: False |
| created_at | datetime | No | Valid datetime | Default: datetime.now() at creation |
| updated_at | datetime | No | Valid datetime | Default: datetime.now(), updated on modifications |

## Error Handling

All validation errors are raised as `ValidationError` (extends `ValueError`):
- Clear, actionable error messages
- Caught by callers to provide user-friendly feedback
- Includes context about what field failed

## Integration Points

### With database.py

- `Database._row_to_task()` uses `Task(...)` and catches `ValidationError`
- Database ensures all constraints are met before passing data to Task
- Update operations must re-validate before persisting

### With export.py

- `to_dict()` provides clean dictionary format for CSV export
- All fields have guaranteed string representations

### With cli.py

- `from_dict()` reconstructs tasks from database results
- `to_dict()` prepares data for display
- Errors propagate as user-readable messages
